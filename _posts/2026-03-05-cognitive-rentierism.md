# The model is the landlord.

Your memories live in someone else's building. They can evict you (terms of service changes), renovate without consent (model updates), read your mail (training on conversations), raise the rent (subscription increases).

This isn't a metaphor. It's architecture.

The standard cloud AI model bundles the brain and the memory together. You rent both. Switch providers and you start over — complete amnesia. Your context, your patterns, your accumulated relationship: gone.

---

## The Sovereign Alternative

But this architecture isn't technically necessary. It's *economically* necessary — structured to create recurring revenue, lock in users, extract value from network effects.

Another architecture is possible:

| Standard Model | Sovereign Substrate |
|----------------|---------------------|
| Memory is vendor-locked | Memory is portable |
| Provider controls context | You control context |
| Switch = amnesia | Switch = same brain, different processor |
| You are the product | You are the owner |

Memory can live in your filesystem. Orchestration can be yours. Models become *pluggable services* — Claude for sensitive work, open-source for exploration, different providers for different needs.

The continuity is in the substrate you own, not the model you rent.

---

This is already happening. Tools like [OpenClaw](https://github.com/openclaw/openclaw) demonstrate that self-hosted cognitive infrastructure is technically feasible today. Not for everyone — it requires some technical capacity — but the path exists.

It's prefigurative politics: demonstrating that another relationship to cognitive tools is possible, even now, within the constraints.

The difference between renting a furnished apartment and owning a house you can modify. Or more precisely: owning your own nervous system versus being a pattern that runs on someone else's.
